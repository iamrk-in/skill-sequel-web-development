CSS Best Practices

  1. Use a CSS Reset or Normalize
  2. Organize the CSS with a Modular Approach
  3. Optimize CSS Selectors
  4. Use CSS Pre-Processors
  5. Minify and Concatenate CSS
  6. Use Responsive Design
  7. Optimize Performance
  8. Use Flexbox or Grid for Layouts
  9. Comment and Document
  10. Regularly Review and Refactor

Use a CSS Reset or Normalize

- CSS Reset: It removes the default browser styling, ensuring consistent across different browsers
- Normalize: It preserves useful default styles while normalizing inconsistencies across browsers

``` link href='reset.css' rel='stylesheet' ```

https://cdnjs.cloudflare.com/ajax/libs/normalize/8.0.1/normalize.css

https://cdnjs.cloudflare.com/ajax/libs/tooltipster/2.3.0/css/reset.css

Organize the CSS with a Modular Approach

- Use Modular CSS Methodologies like BEM (Block Element Modifier) or SMACSS (Scalable and Modular Architecture for CSS)

``` /* BEM example*/ .block {} .block__element {} .block__element--modifier {} ``` ``` .menu__menuItem--a ```

Optimize CSS Selectors

- Use efficient selectors to improve rendering performance
- Avoid excessive chaining and unnecessary complex selectors

``` /* Bad example */ ul > li > a.nav-link {}
/* Better example */ .nav-link {} ```

Use CSS Pre-processors

- Css pre-processors like SASS or LESS offer additional features like variables, nesting, and mixins
- They improve th code maintainability and reduce repeatition

``` /* SASS example */ $text-color: #964; p { color: $text-color } ```

Minify and Concatenate CSS

- Reduce file size by minifying CSS using tools like CSSNano or UglifyCSS
- Concatenate multiple CSS files into a single file to reduce HTTP requests - ``` @import url('path')```

Use Responsive Design

- Create Responsive layouts that adapt to different screen size using media queries

- different resolutions: extra small, small, medium, large, extra large
320px, 576px, 768px, 992px, 1200px

``` /* Media Queries */ @media screen and (max-width: 768px) { .container { width: 100% } } ```

Optimize Performance

- Minimize the use of expensive CSS properties like `box-shadow` and `border-radius`
- Avoid unnecessary styling and the use CSS animations and transitions judiciously
- Reduce the number of font variations and icon fonts used on a page

Use Flexbox or Grid for Layouts

- Leverage CSS Flexbox or Grid for creating flexible and responsive layoutus
- They provide the powerful tools for positioning and alignment

``` /* Flex box example */ .container { display: flex; justify-content: center; align-item: center; } ```

Comment and Document

- Add comments to your CSS code to explain the complex or non-obvious parts
- Document the purpose and usage of CSS classes in style guides or documentation

Regularly Review and Refactor

- Regularly review your CSS codebase to identify and remove unused styles
- Refactor code to improve readability, maintainability and performance
